## Findings (all minor; ranked)

### 1. Benign 404 console errors during pool creation — cosmetic
`ensureFunded()` calls `horizon.loadAccount()` on the 4 fresh accounts before Friendbot funds
them; each returns **404** and is correctly caught, but it logs 4 red errors in devtools.
Nothing breaks. **Fix (optional):** probe account existence without triggering a logged 404
(e.g. `friendbot` first, or fetch and treat 404 as "not found" via a quiet endpoint).
_Files: `apps/web/src/lib/livepool.ts` (`ensureFunded`)._

### 2. Transient balance mismatch right after create-pool — minor UX
Immediately after deploy, the hero showed **₱1,600** (client-computed `contributions − spends`)
while the "On-chain treasury" card showed **1,000 USDC** (a stale `get_balance` read from
mid-deploy). It self-corrected to **1,600 USDC** within ~15 s (the card's refetch interval). No
funds lost. **Fix:** invalidate the `live-treasury` query in `useCreatePool` `onSuccess` (and
after writes) so both cards refresh together, or drive both from one query.
_Files: `apps/web/src/hooks/usePool.ts`, `apps/web/src/components/LiveOnChain.tsx`._

### 3. Recipient shows as raw address, not name — minor UX
A pending/completed spend's title reads **"₱1,200 → GCHX…7FAW"** instead of "→ MVP Sports
Depot". The friendly name lives only in the memo (recipient names aren't stored on-chain), so
`readPool` uses the short address for the title. The AI *does* surface the name from the memo.
**Fix:** show the payee name in the title — parse it from the memo, or keep a client-side
address→name map.
_Files: `apps/web/src/lib/livepool.ts` (`readPool` → `recipientName`), `apps/web/src/routes/Spend.tsx`._

### 4. Setup "Parse" shows "Snacks" but the live pool uses "Refreshments" — minor consistency
The Rules textarea example says "snacks up to ₱1,500", so the (cosmetic) parsed policy shows a
**Snacks** category, while the deployed pool's third category is **Refreshments**. The card's
footer ("Your live pool enforces … these category caps") implies they match. **Fix:** change the
example text to "Refreshments", or have the parsed card read the live categories.
_Files: `apps/web/src/routes/Setup.tsx` (EXAMPLE)._
